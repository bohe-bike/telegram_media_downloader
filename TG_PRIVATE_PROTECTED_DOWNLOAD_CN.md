# 私有群/受限资源下载用法

本文说明如何通过 Telegram Bot 命令下载已经加入的私有群、私有频道资源，以及如何处理禁止转发/受保护内容。

## 前提条件

1. 运行本项目的 Telegram 用户账号必须已经加入源私有群或私有频道。
2. 发命令的 Telegram 用户必须在 `config.yaml` 的 `allowed_user_ids` 中。
3. 使用的是消息链接，不是群邀请链接。

示例配置：

```yaml
allowed_user_ids:
- 'me'
```

## 下载私有群或私有频道资源

向 Bot 发送：

```text
/download 源消息链接 起始消息ID 结束消息ID [过滤条件]
```

从第 1 条消息下载到最新消息：

```text
/download https://t.me/c/12000000/123 1 0
```

下载指定范围：

```text
/download https://t.me/c/12000000/123 100 200
```

只下载视频：

```text
/download https://t.me/c/12000000/123 1 0 media_type == 'video'
```

其中 `结束消息ID` 为 `0` 表示一直处理到最新消息。

## 下载单条消息

可以直接把消息链接发送给 Bot：

```text
https://t.me/c/12000000/123
```

Bot 会尝试下载这一条消息中的媒体内容。

## 转发受保护内容

如果源群或频道开启了禁止转发/保护内容，普通转发可能失败。可以使用 `/forward`，项目会尝试通过“下载后重新上传”的方式转发到目标聊天。

向 Bot 发送：

```text
/forward 源消息链接 目标消息链接 起始消息ID 结束消息ID [过滤条件]
```

示例：

```text
/forward https://t.me/c/12000000/123 https://t.me/c/13000000/1 1 0
```

只转发视频：

```text
/forward https://t.me/c/12000000/123 https://t.me/c/13000000/1 1 0 media_type == 'video'
```

## 常见注意事项

- 源私有群/频道必须是当前登录的用户账号已经加入的聊天。
- Bot 自身不一定需要加入源群，但必须能接收你的命令。
- 私有群链接通常形如 `https://t.me/c/.../...`。
- 如果链接解析失败，请先确认该账号能在 Telegram 客户端中打开对应消息。
- 如果下载范围很大，建议先用较小范围测试命令是否正常。
